# Open data

Static, versioned, CORS-open data files served from GitHub Pages — a plain-GET API
(no auth, no key). An OpenAPI description is at [`../api/openapi.yaml`](../api/openapi.yaml).

**License:** curated data is **CC BY 4.0** ([LICENSE](LICENSE)). Third-party analyst
sources cited per record retain their own terms — each carries a `provenance` field;
`analyst` rows are pointers, not relicensed. Suggested attribution:
*"US AI Infrastructure Monitor — data CC BY 4.0, https://vijay-sachdeva.github.io/us-ai-infra"*.

## Files

| File | What | Format |
|---|---|---|
| `projects.json` | **Canonical** named project + power-deal ledger | JSON ([schema](../schemas/projects.schema.json)) |
| `projects.csv` | The ledger, flat (one row per campus) | CSV |
| `projects.geojson` | The ledger as geocoded points | GeoJSON |
| `projects.parquet` | The ledger, columnar (same columns/row order as the CSV) | Parquet |
| `sources.json` | Source ledger: every cited label → verified URL + provenance | JSON |
| `grid.json` | Grid headroom by balancing authority (derived, EIA-930 + EIA-860) | JSON |
| `power_econ.json` | Industrial power price by state (EIA-861) | JSON |
| `queues.json` | Generation interconnection queue by ISO (LBNL Queued Up + 78% withdrawal haircut) | JSON |
| `siting.json` | Per-state modeled white-space siting score (0–100) | JSON |
| `datapackage.json` | [Frictionless](https://datapackage.org) Data Package descriptor — Table Schema for `projects.csv` + every resource | JSON |
| `catalog.jsonld` | [DCAT 3](https://www.w3.org/TR/vocab-dcat-3/) catalog — datasets + distributions metadata for harvesters | JSON-LD |

`projects.csv` / `projects.geojson` / `projects.parquet` are generated from `projects.json`. The four metric
feeds are generated by `scripts/` via `.github/workflows/refresh-data.yml` — do not edit
by hand. The front-end (`index.html` → `hydrate()`) fetches all of these on load and
merges them into `DATA`; feeds are **additive and degrade gracefully** (a missing or
failed feed leaves the curated `DATA` in place and never blanks a chart).

## Provenance model (per project record)

Three orthogonal fields instead of a single "tier":

- **`provenance`** — where it came from: `government` · `filing` · `operator` · `trade_press` · `analyst`.
- **`transformation`** — how the value relates to the source: `reported` · `calculated` · `estimated` · `forecast` · `scenario`.
- **`confidence`** — `high` · `medium` · `low`.

Each `sources[]` entry was opened and checked: **`supports_claim: true`** means the page
was confirmed to state the operator + ~MW + status. Several headline MW are announced/
ultimate targets (see `capacity_type`, `transformation`, `note`), not confirmed draw.

## "fetched" vs "observed"

The metric feeds carry `lastUpdated` = when CI last **fetched** them. The underlying public
datasets can report an earlier observation period (EIA-861 publishes with a lag; LBNL is
annual). Treat `lastUpdated` as retrieval, not observation.

## Versioning

`projects.json.version` is bumped on schema/record changes; see [`../CHANGELOG.md`](../CHANGELOG.md)
and [`../CITATION.cff`](../CITATION.cff). Cite a specific build by its version + retrieval date.

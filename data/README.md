# Open data

Static, versioned, CORS-open data files served from GitHub Pages ‚Äî a plain-GET API
(no auth, no key). An OpenAPI description is at [`../api/openapi.yaml`](../api/openapi.yaml).

**License:** curated data is **CC BY 4.0** ([LICENSE](LICENSE)). Third-party analyst
sources cited per record retain their own terms ‚Äî each carries a `provenance` field;
`analyst` rows are pointers, not relicensed. Suggested attribution:
*"US AI Infrastructure Monitor ‚Äî data CC BY 4.0, https://vijay-sachdeva.github.io/us-ai-infra"*.

## Files

| File | What | Format |
|---|---|---|
| `projects.json` | **Canonical** named project + power-deal ledger | JSON ([schema](../schemas/projects.schema.json)) |
| `projects.csv` | The ledger, flat (one row per campus) | CSV |
| `projects.geojson` | The ledger as geocoded points | GeoJSON |
| `projects.parquet` | The ledger, columnar (same columns/row order as the CSV) | Parquet |
| `sources.json` | Source ledger: every cited label ‚Üí verified URL + provenance | JSON |
| `grid.json` | Grid headroom by balancing authority (derived, EIA-930 + EIA-860) | JSON |
| `power_econ.json` | Industrial power price by state (EIA-861) | JSON |
| `queues.json` | Generation interconnection queue by ISO (LBNL Queued Up + 78% withdrawal haircut) | JSON |
| `siting.json` | Per-state modeled white-space siting score (0‚Äì100) | JSON |
| `history/*.csv` | **Time-series archive** ‚Äî each feed's daily snapshots accumulating over time (tidy/long CSV); see [`history/README.md`](history/README.md) | CSV |
| `datapackage.json` | [Frictionless](https://datapackage.org) Data Package descriptor ‚Äî Table Schema for `projects.csv` + every resource | JSON |
| `catalog.jsonld` | [DCAT 3](https://www.w3.org/TR/vocab-dcat-3/) catalog ‚Äî datasets + distributions metadata for harvesters | JSON-LD |

`projects.csv` / `projects.geojson` / `projects.parquet` are generated from `projects.json`. The four metric
feeds are generated by `scripts/` via `.github/workflows/refresh-data.yml` ‚Äî do not edit
by hand. The front-end (`index.html` ‚Üí `hydrate()`) fetches all of these on load and
merges them into `DATA`; feeds are **additive and degrade gracefully** (a missing or
failed feed leaves the curated `DATA` in place and never blanks a chart).

## Provenance model (per project record)

Three orthogonal fields instead of a single "tier":

- **`provenance`** ‚Äî where it came from: `government` ¬∑ `filing` ¬∑ `operator` ¬∑ `trade_press` ¬∑ `analyst`.
- **`transformation`** ‚Äî how the value relates to the source: `reported` ¬∑ `calculated` ¬∑ `estimated` ¬∑ `forecast` ¬∑ `scenario`.
- **`confidence`** ‚Äî `high` ¬∑ `medium` ¬∑ `low`.

Each `sources[]` entry was opened and checked: **`supports_claim: true`** means the page
was confirmed to state the operator + ~MW + status. Several headline MW are announced/
ultimate targets (see `capacity_type`, `transformation`, `note`), not confirmed draw.

## "fetched" vs "observed"

The metric feeds carry `lastUpdated` = when CI last **fetched** them. The underlying public
datasets can report an earlier observation period (EIA-861 publishes with a lag; LBNL is
annual). Treat `lastUpdated` as retrieval, not observation.

## Versioning

`projects.json.version` is bumped on schema/record changes; see [`../CHANGELOG.md`](../CHANGELOG.md)
and [`../CITATION.cff`](../CITATION.cff). Cite a specific build by its version + retrieval date.

## sec_filings.json (generated daily)

Latest disclosure-relevant SEC filings (10-K/10-Q/8-K; 20-F/6-K for foreign filers) per tracked player, from the EDGAR submissions API via scripts/fetch_sec_filings.py ‚Äî primary metadata with direct document links. Feeds the Players tab Filings-watch module, which flags periodic filings dated after the commitment book last-review stamp.

## connections.json (hand-curated)

The connect-the-dots layer ‚Äî verified edges between facts already published elsewhere on the dashboard (no new figures). Each connection carries a **connectionTier** (`both_sides` / `one_side` / `inference`) ‚Äî the cross-document *link* confidence, distinct from each source''''s own primary/analyst/modeled tier ‚Äî so an unverified link cannot render untiered (enforced by schemas/connections.schema.json + tests/test_data_quality.py). Curated by hand and honours the same 150-day staleness gate; the daily refresh only *proposes* candidates for human verification. Powers the "Connecting the dots" module on Overview.
